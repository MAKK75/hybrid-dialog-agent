Программный комплекс гибридного мультимодального диалогового агента, разработанный для работы в условиях строго ограниченных вычислительных ресурсов (встраиваемые платформы уровня NVIDIA Jetson Orin Nano). Проект создавался в рамках магистерской диссертации (ВКР) в НИУ ВШЭ, ориентирован на использование в роботах-ассистентах космического базирования (проект "Белка").

## Особенности проекта

* **Гибридная архитектура:** Разделение вычислений на локальные (зрение, STT, TTS) и облачные (LLM, VLM) контуры для обхода ограничений VRAM (менее 4 ГБ).
* **Асинхронная оркестрация:** Многопоточная модель с неблокирующей очередью задач (5 независимых потоков: камера, зрение, аудио, мониторинг системы, принятие решений).
* **Локальное зрение (Offline):** Детекция лиц (YuNet) и распознавание/верификация (SFace) через OpenCV (ONNX) без передачи биометрии в сеть (30 FPS).
* **Адаптивная долгосрочная память:** Интеграция фреймворка **Mem0** и локальной **ChromaDB**. Агент запоминает факты о собеседниках и контекстно персонализирует диалог.
* **Локальный речевой контур:** Распознавание речи через оптимизированный `faster-whisper` и высококачественный синтез через `Silero TTS` с кастомным пайплайном нормализации текста и числительных.
* **Отслеживание внимания (Active Speaker Tracking):** Адаптивное переключение контекста между несколькими собеседниками в кадре на основе трекинга лиц (IoU) и речевых триггеров.

## Эволюция архитектуры (Структура репозитория)

В репозитории представлены 3 этапа разработки системы (папка `src/`):

1. **`v1_local_sync` (Прототип 1):** Полностью локальный синхронный агент на базе VLM Qwen2-VL-2B (4-bit). Демонстрирует перегрузку VRAM и задержки из-за блокировок потока.
2. **`v2_hybrid_async` (Прототип 2):** Переход на гибридную многопоточную модель. Локальное зрение + облачный YandexGPT + облачный синтез речи (edge-tts).
3. **`v3_final` (Финальная версия):** Оптимизированный релиз. Локальный Silero TTS, Mem0 для памяти, YandexGPT (LLM), Qwen 3.6-VL для визуального анализа. Максимальная защита биометрии и минимальные задержки.

## Установка и запуск (Финальная версия)

### Требования
* Python 3.10+
* Оперативная память: от 8 ГБ
* VRAM: от 4 ГБ 
* Аккаунт Yandex Cloud с доступом к Foundation Models API.

### Шаг 1. Клонирование и зависимости
```bash
git clone https://github.com/MAKK75/hybrid-dialog-agent.git
cd hybrid-dialog-agent/v3_final
pip install -r requirements_ver3.txt
```

### Шаг 2. Скачивание весов моделей (ONNX)
Поместите в папку `v3_final` следующие модели компьютерного зрения:
* `face_detection_yunet_2023mar.onnx`
* `face_recognition_sface_2021dec.onnx`

*(Модели можно найти в [OpenCV Model Zoo](https://github.com/opencv/opencv_zoo))*

### Шаг 3. Настройка API ключей
Создайте файл `.env` (или укажите ключи прямо в коде в классе `Config`, но **не коммитьте их в Git**):
```python
class Config:
    YANDEX_FOLDER_ID = "ваш_folder_id"
    YANDEX_API_KEY = "ваш_api_key"
```

### Шаг 4. Запуск
```bash
python agent_ver_3_fix.py
```

## Сценарии работы
* **Регистрация:** При появлении нового лица система запросит имя и сохранит 128-мерный эмбеддинг в локальную ChromaDB.
* **Долгосрочная память:** Скажите агенту "Я люблю пить черный чай по утрам". В фоновом режиме факт будет извлечен LLM и сохранен. Спросите "Что ты помнишь обо мне?", и система использует эти данные для ответа.
* **Зрение по запросу:** Спросите "Что ты видишь?". Агент сделает снимок, сожмет его, отправит в облачную VLM Yandex Cloud и озвучит описание обстановки с упоминанием людей в кадре.
* **Поиск:** Скажите "Найди в интернете [запрос]". Агент сходит в Yandex Search API, очистит Markdown-разметку и прочитает ответ.
